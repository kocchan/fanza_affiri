# CLAUDE.md — このプロジェクトの作業ルール

Claude が毎回読む運用ルール。詳細な構成は [README.md](README.md) を参照。

## このプロジェクトについて
- DMM/FANZAアフィリエイトの**X運用支援**（コンテンツ生成〜運用ナレッジ）。実際のX投稿はしない（候補を作るだけ／投稿は人が手動）。
- **git リポジトリではない**。コミット/PR/Conventional Commits/TDD は前提にしない（コード開発プロジェクトではない）。
- 利用者は非エンジニア。専門用語を避け、結論と次アクションを先に出す。

## TODO 管理（2層モデル）
TODO は役割の違う2つを使い分ける。

1. **`TODO.md`（長期の“やりたいこと台帳”）** … セッションを跨いで残る。`🔄進行中 / ⬜これからやる / ✅完了` で管理。
2. **内蔵タスクリスト（TaskCreate/TaskUpdate）** … その作業中だけの実行ステップ可視化。消えてよい。

### 守るルール
- **作業開始時：まず `TODO.md` を見る**。「次のTODOやって」と言われたら上から着手する。
- **2ステップ以上の作業は、内蔵タスクリストに分解してから進める**（進捗を可視化）。
- **項目を終えたら必ず `TODO.md` を更新する**：`[ ]`→`[x]` にし「✅完了」へ移動、やったことを1〜2行で要約（参照したファイル/スクリプト名も書く）。
- **その場でやらない新要望は `TODO.md`「これからやる」に追記**してから先へ進む（取りこぼし防止）。
- 1項目 = 「**タイトル** … 内容」の2行構成で揃える（`TODO.md` 冒頭の凡例に従う）。

## ナレッジ・スキルの追従
- **学び/フィードバックが出たら、その場で該当ナレッジ（`ナレッジ/…`）に反映する**（例：大喜利の勝ち筋は `ナレッジ/集客・導線/…大喜利（坊主リプ集客）.md` へ）。座学だけにせず実例を足す。
- スキルを足す/直すと **README のスキル一覧は Stop フックで自動更新**される（`.claude/sync_skills_readme.py`）。手で編集しない。
- スキル本体の文面・手順を変えたら、対応する**元ナレッジも一緒に直す**（スキルはナレッジを参照して動くため）。
