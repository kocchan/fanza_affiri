# 副業：DMM アダルトアフィリエイト（X 運用）

## 1. 事業概要

X（旧 Twitter）上で DMM のアダルト作品をアフィリエイト紹介し、リンク経由の購入で報酬を得るビジネス。

- 収益の本質は **アフィリエイトリンクをいかに多くのユーザーに踏ませるか**。
- 最大の障壁は **X のシャドウバン**。運用の本質は「いかにシャドウバンを回避し続けるか」にある。
- 集客役の **メインアカウント** と、リンクを流す **サブアカウント** を役割分担し、複数運用でリスクを分散する。
- コンテンツは**競合パクリ中心**（伸びている競合の投稿を参考に同型で作る）。
- 完全自動化は難易度が高いため、当面は**手動運用**が基本。

### 当面のゴール
まずは **検証フェーズ（収益度外視）**。
「シャドウバンを回避しながらメイン → サブの導線が機能するか」を確かめることが目的。

---

## 2. 収益・導線の仕組み

```
メインアカウント（集客）
  └─ 露出控えめ＋文字でそそる投稿で人を集める
        └─ サブアカウントがメイン投稿へリプライ
              └─ リプライ内のアフィリエイトリンク
                    └─ ユーザーが DMM で購入 → 報酬発生
```

- メインには直接リンクを貼らず、**サブがメイン投稿へリプライする形で URL を届ける**。
- これによりメインの信用度を守りつつ、リンク露出はサブに肩代わりさせる。

---

## 3. 運用方法（2フェーズ）

運用は「①システムが作る」→「②人が貼る」の2フェーズに分ける。

### フェーズ① システムフェーズ（コンテンツ作成）
ツール `fanza_auto/fetch_and_build.py` が自動で実行する。

- **1日2投稿分**のコンテンツをまとめて生成する。
- 各投稿について、**メインとサブ両方のアカウントの投稿内容**を考える。
  - メイン：集客用の本文＋使う画像/動画（規制セーフな素材）
  - サブ：メイン投稿へぶら下げるアフィリエイトリンク付きリプライ
  - （ブースター：弱めの盛り上げコメント案）
- 生成物はファイルに格納する（`output/<日付>/投稿シート.md` と素材一式）。

### フェーズ② 人の運用フェーズ（投稿）
人は「考える」必要がなく、**システムが作った内容をコピーして貼るだけ**。

1. メインアカウントを開き、投稿#1の本文＋素材をコピーして投稿
2. サブアカウントを開き、その投稿へリプライでアフィリエイトリンクを貼る
3. （ブースターで時間をあけて弱く盛り上げる）
4. 2投稿目（#2）も同様に行う

> ポイント：判断（作品選定・文章・規制チェック）はフェーズ①に寄せ、
> フェーズ②は手作業（＝シャドウバン回避のため自動化しない部分）に徹する。

---

## 4. 複数アカウント運用の必要性（リスク分散の考え方）

リスク分散の方法を整理すると、以下の結論になる。

| 考え方 | 評価 |
| --- | --- |
| フォロワーの分散でリスク分散する | ❌ そもそも**フォロワーは必要ない** |
| 信用度の高いアカウントを複数運用してリスク分散する | ❌ やろうとしている方法だと**信用度も分散**してしまう |
| メインを守り、サブに URL を流させる | ✅ 採用 |

### 採用する運用方針
- **メインアカウントをシャドウバンされないように守る。**
- アフィリエイト **URL はサブアカウントのリプライから流す**。
- **メインも複数運用**しておくとリスク分散になって安全。
- シャドウバンされていないとき、**メイン同士で相互リツイート**しておくとリスク分散になる。

---

## 5. アカウントの役割

### メインアカウント（集客・信用維持）
- **スマホからログイン**する
- 肌の露出が少ない ＋ 文字でそそられる投稿をする（競合を調査して型を作る）
  - 集客投稿の合間に、**文字だけ・無リンクの“そそる”つぶやき**を不定期に挟んで人間味・信用を育てる。
    → 型は [X運用ナレッジ_文字投稿（そそる）.md](../ナレッジ/コンテンツ/X運用ナレッジ_文字投稿（そそる）.md)、生成はスキル `x-tease-posts`
- 信用度を常に高い状態で維持する
- アフィリエイトリンクは直接貼らず、**サブのリプライから流す**

### サブアカウント（リンク発信）
- **PC からログイン**する
- アフィリエイトリンクの作成・発信用
- メイン投稿へリプライして URL を届ける
- 信用度は低くなってしまう前提で運用する

- 自動化するなら **情報収集（競合調査）** と **アカウント作成** の部分に絞る。
