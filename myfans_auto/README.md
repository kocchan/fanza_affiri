# MyFans 投稿セット・ダッシュボード

MyFansの投稿を、FANZA（`fanza_auto/`）とは**別の専用ボード**で管理する。
DMMのようなアフィリエイトAPIが無く、投稿ごとに個別のアフィリンクが要る——という前提の違いが
あるため、意図的に `fanza_auto` とは切り離してある。

## 仕組み

```
MyFansの投稿URL（自分が発行したアフィリンク）
   │
   └─ myfans_fetch.py
         ① requests で公開OGPメタタグ（og:title/image）を読む（軽量・ログイン不要）
         ② Playwrightで実ページを開き、年齢確認「はい」だけクリックして
            本文全文（DOM上の実テキスト）と無料公開サンプル動画（HLS）を取得
            → ffmpegで sample.mp4 として保存（②が失敗しても①の結果で続行）
   │
   └─ 1作品＝1フォルダで works/ 直下に出力
         works/<投稿ID>_<作品名>/
            ├─ 01.jpg          … サムネ画像（og:image）
            ├─ item.json       … 作品情報（タイトル全文・本文全文・投稿者・アフィリンク・アーカイブ状態）
            ├─ 投稿内容.md      … 参照用メモ（本文全文つき）
            └─ sample.mp4      … 無料公開サンプル動画（自動取得。無ければ空欄）
         works/board.html      … 全体ボード（未アーカイブの全作品）
         works/archive.html    … アーカイブ一覧
         works/board_<投稿ID>.html … 1作品ぶんの個別ボード（動画編集・投稿文コピー）
```

## 何を自動化していて、何をしていないか

- **年齢確認への「はい」クリックだけ自動化している**。これは誰でも通す標準的な導線で
  ボット対策ではないので、実ブラウザ（Playwright）で開いて素通りしている。
- 取得している動画は、投稿ページで**ログイン・購入なしに誰でも再生できる無料サンプル**
  （HLS配信・`content.mfcdn.jp`の公開CDN）だけ。FANZAの`sample.mp4`と同じ位置づけの
  公式プレビュー素材で、購入者限定の本編には一切アクセスしていない。
- **ログインは自動化しない**。購入後の本編動画などログインが要る情報は取りに行かない方針。
  一度Playwrightでログイン自動化を試したが、ログイン画面のCloudflare Turnstileが自動化
  ブラウザを検知して失敗した。これを回避する技術（フィンガープリント偽装等）は
  プラットフォームが意図的に設置した自動化対策を突破する行為にあたるため実装しない。
- **本編（購入後のフル動画）を使いたいとき**は、人がChrome拡張機能で保存して
  プロジェクトのルートフォルダに置く運用にする（②③の手順）。ボード側は
  「🎬 動画を取り込む」ボタンで、タイトルの一致から対応フォルダへ自動で振り分け、
  既にある `sample.mp4` を上書きする。

## 使い方

```bash
# プロジェクトのルートフォルダで実行
python3 myfans_auto/scripts/serve.py           # works/board.html を開く（既定ポート8000）
python3 myfans_auto/scripts/serve.py 8100       # ポートを指定（FANZAのボードと同時に開く場合など）
```

### ① 投稿を取り込む（タイトル全文・本文全文・サムネ・サンプル動画まで自動）

ボード上部のURL欄に、**自分がMyFansで発行したアフィリンク**
（例: `https://link.affiliate.myfans.jp/r/<コード>`）を貼って「＋取り込む」。数秒〜十数秒かかる。

- MyFansのアフィリンクは**クリエイターごとに別のコード**が発行される（DMMの`af_id`のように
  口座全体で1つ固定ではない）。素のURL（`myfans.jp/posts/...`だけで`aff=`が無いもの）を貼ると
  成果が計測されないので注意。
- 取り込みは `myfans_fetch.py` が裏で動く。CLI単体でも実行できる：
  ```bash
  python3 myfans_auto/scripts/myfans_fetch.py <URL>
  ```
- 既に取り込み済みの投稿IDはスキップされる（差分追加）。
- サンプル動画が取れなかった投稿（サンプルを設定していない等）は、②③の手順で動画を追加できる。
- 本文の自動抽出はDOM構造に依存するため、MyFansの改修で取れなくなる可能性はある。その場合は
  `myfans_fetch.py <URL> --description "<本文>"` で手動指定できる（CLIのみ・保険用）。

### ②（任意）本編の動画を使う

1. Chrome拡張機能（FetchV Video Download / ストリームレコーダー等）を入れる。
2. いつも通りMyFansにログインして該当の投稿を再生し、拡張機能で動画を保存する。
3. 保存した動画ファイルを、**そのままプロジェクトのルートフォルダ**（このリポジトリ直下）に置く。
   ファイル名はブラウザのタブタイトルのままでよい
   （例：「【〇〇】…タイトル… ◯◯さんのプライベートSNS myfans マイファンズ.mp4」）。

### ③（任意）動画を取り込む

①のサンプル動画で足りる場合はこの手順は不要。本編を使いたいときだけ、ボード上の
「🎬 動画を取り込む」ボタンを押す。裏で `import_video.py` が：

- ルートフォルダの動画ファイル名と、`works/` 内各作品の `item.json` の `title`/`creator` を
  正規化して突き合わせる（ダウンロード拡張機能はタブタイトルをそのままファイル名にすることが多く、
  それはMyFansのog:titleと同じ文字列なので前方一致で高精度にマッチする）。
- **候補が1件に絞れたときだけ**自動で `sample.mp4` として作品フォルダへ移動する。
- 候補が複数（あいまい）・0件（対応する取り込み済み作品が無い）のときは移動せず、結果を画面に表示する。
  その場合は手動で対象フォルダに `sample.mp4` という名前でファイルを置けば、次に開いたときから
  同じように扱われる。
- CLI単体でも実行できる：
  ```bash
  python3 myfans_auto/scripts/import_video.py
  ```

動画が入った作品は、以降 `fanza_auto` と**まったく同じ**切り抜き・画面トリミング・サムネ作成機能が使える
（`cut_video.py`/`crop_video.py`/`grab_frame.py`/`crop_image.py` は `fanza_auto/scripts/` からの
コピー。中身は完全に独立しているので `fanza_auto` 側の変更の影響は受けない）。

### ④ 投稿作業

各カードに「①リンク投稿（アフィリンク入り）」「②メイン投稿（①を引用して出す）」のコピー欄がある。

- **①リンク投稿**：`fanza_auto/scripts/post_text.py`/`templates.py` をそのまま流用（リンクを
  【】で囲んだ短い誘導文）。
- **②メイン投稿**：`caption.py`（MyFans専用）が本文（description）から「場所・シチュエーション」
  だけを許可リストで拾い（エアビー・お泊まり・ホテル・合コン・デート等。行為語やハッシュタグは
  絶対に使わない）、短い"あるある反応"文にする（例：「ノリでエアビーした結果がこれwww」）。
  該当するシチュエーションが本文に無いときだけ、`templates.py`の固定リアクション文
  （`REACTION_HOOKS`）にフォールバックする。

**メイン投稿は個別ページで自由に編集できる**（テキストエリアに直接書き換え）：
- **💾 保存**：編集した文章をそのまま `works/posts.json` に保存する。個別ページ・全体ボード
  両方に即反映される（保存時に両方を再生成する）。
- **文章を作り直したいとき**：ボード上にボタンは無い。Claude Codeのチャットで
  「〇〇の投稿のキャプション作り直して」のように直接指示する
  （ボードのボタンからClaudeをその場で呼び出すことはできないため。`caption.py`の
  自動生成はあくまで①の取り込み時点の初期値づけ）。

投稿し終えたら「📦 アーカイブ」でボードから外す（`works/archive.html` に移る）。
アーカイブ一覧では「🗑 完全削除」（フォルダごと消す・元に戻せない）か「↩ 全体ボードに戻す」を選べる。

### 1作品だけを動画つきで開く

ボードの「🎬 動画・画像を編集する →」から個別ページ（`board_<投稿ID>.html`）に移動できる。
直接作るときは：

```bash
python3 myfans_auto/scripts/board.py <投稿ID>
```

切り抜き等の操作自体は①で起動した `serve.py` が処理する（`board_<投稿ID>.html` を file:// で
直接開いた場合は再生・保存のみ）。

## FANZA版との違い（意図的に持たない機能）

| 機能 | FANZA (`fanza_auto`) | MyFans (`myfans_auto`) |
|---|---|---|
| 取り込み元 | DMMアフィリエイトAPI（公式） | 投稿ページの公開OGPメタタグ（非公式・限定情報） |
| 自動取得できるもの | 画像・サンプル動画・評価・ジャンル等 | タイトル全文・本文全文・サムネ画像・無料サンプル動画 |
| 動画 | 自動ダウンロード（本編扱いのサンプル） | 無料サンプルは自動DL。本編が要るときだけ人が拡張機能で保存→取り込みボタンで振り分け |
| MissAV確認 | あり（FANZAの品番流出チェック） | 概念が無いため無し |
| 評価★・ジャンル・発売日・メーカー表示 | あり | 元データが無いため無し |
| 未成年連想ジャンルの自動スクリーニング | あり（`safe_criteria.py`） | 元データが無いため無し。**取り込んだ作品は人が目視で確認する** |
| 投稿文の型選択 | 評価・ジャンル・発売日から自動選択 | 実データが無いため常に固定リアクション文 |

## スクリプト早見

| スクリプト | 役割 |
|---|---|
| `myfans_fetch.py` | MyFansの投稿URLからタイトル全文・本文全文・サムネ・サンプル動画を取得し `works/` 直下に追加 |
| `import_video.py` | プロジェクトのルートフォルダの動画を、タイトル一致で対応フォルダへ `sample.mp4` として振り分け |
| `board.py` | 投稿ID を渡して1作品だけの動画つきボード `board_<投稿ID>.html` を作る。`--all` で全作品＋全体ボード一括再生成 |
| `dashboard.py` | `works/board.html`（全体ボード）と `works/archive.html`（アーカイブ一覧）を作る |
| `serve.py` | 全体ボードを、切り抜き・アーカイブ・URL取り込み・動画取り込みも使える状態で開く |
| `cut_video.py` / `crop_video.py` / `grab_frame.py` / `crop_image.py` | 動画・画像の切り抜き（`fanza_auto/scripts/` からのコピー・完全に独立） |
| `common.py` | works/ 走査・item.json 読み書きの共通土台（DMM関連の設定は持たない軽量版） |
| `caption.py` | 本文（description）から場所・シチュエーションを許可リストで拾い、メイン投稿の短い反応文を作る |

## 安全・運用上の注意

- 未成年に見える/制服強調のコマは人が目視で必ず除外する（Xのゼロトレランス）。
- センシティブ設定は必ずONにする。
- リンクは①リンク投稿に持たせ、②メインはそれを引用する（メイン本体にリンクを貼らない）。
- 取り込むURLは必ず**自分が発行したMyFansアフィリンク**にする。
