# FANZA 投稿セット自動生成ツール

FANZAの人気動画を取得し、「アカウントを開いて貼るだけ」の投稿セットを生成する。
作品ごとにフォルダを作り、**全作品を1枚の投稿ボード（`works/board.html`）に集約**する。

## 仕組み

```
DMMアフィリエイトAPI（人気動画ランキング）
   │
   ├─ すでに works/ にある作品(cid)はスキップ＝未掲載の新作だけを仕上げる（差分追加）
   ├─ 動画からの規制セーフフレームを優先して画像を採用（公式サンプルは補充）
   ├─ サンプル動画(mp4)もフォルダに同梱
   │
   └─ 1作品＝1フォルダで出力（日付フォルダは作らない）
         works/<cid>_<作品名>/
            ├─ 01.jpg 〜          … 投稿に使う採用画像
            ├─ sample.mp4         … サンプル動画（動画優先で投稿）
            ├─ item.json          … 作品情報（評価・収録時間・ジャンル等）
            └─ 投稿内容.md        … 勝ち型を適用した本文一式
         works/board.html         … 投稿ボード（テキスト専用・こちらがメイン）
         works/index.html         … 旧ダッシュボード（画像まわり担当）
```

## 投稿文はどう作られるか

`post_text.py` が `item.json` の**実データ**を読んで、その作品に合った文を1本作る。
以前はテンプレートのランダム選択だけで、どの作品でも同じことを言っていた。

| 作品の状態 | 選ばれる型 |
|---|---|
| ★4.7以上＋レビュー5件以上 | 記録・ランキングの社会的証明 |
| 発売30日以内 | 速報体 |
| 「清楚/黒髪/めがね」×「スタイル系」が両立 | ギャップ提示 |
| ★4.3以上＋3件以上 | 断定・煽り |
| レビューがまだ少ない | 共感・自虐／問いかけ |

本文には評価・収録時間・ジャンルの**具体**を1行入れる
（例：「レビュー★4.91（11件）。この点数はめったに出ない。」）。

> ⚠️ **本文に出してよいジャンルは許可リスト方式**（`post_text.py` の `SAFE_GENRES`）。
> リストに無い語は全部落とすので、露骨語（中出し/顔射/盗撮…）や未成年連想語
> （女子校生/美少女…）が本文に混ざる事故を仕組みで防いでいる。
> 作品としての採否は別レイヤー（`safe_criteria.py` の方針に従う）。
> 該当ジャンルが付いた作品は、ボード上で「画像は要注意」と警告する。

## 使い方（毎日これだけ）

```bash
# プロジェクトのルートフォルダで実行
python3 fanza_auto/scripts/fetch_and_build.py      # 未掲載の新作だけ works/ に追加
python3 fanza_auto/scripts/build_board.py --open   # 投稿ボード（works/board.html）を作って開く
```

**投稿ボード（`works/board.html`）** … テキスト専用の一覧。作品情報・投稿文・アフィリリンクを
ワンクリックでコピーして X に貼る。サーバー不要でダブルクリックでも開ける。

### 1作品だけを動画つきで開く

一覧から作品の cid を決めたら、その1作品だけを**動画つき**で開く：

```bash
python3 fanza_auto/scripts/serve_board.py debz015   # cid を指定（例: debz015）
```

- ブラウザが自動で開き、**動画の再生・切り抜き・保存**ができる。
  - **再生**：シーク（早送り）対応。
  - **✂ 切り抜く**：開始・終了の秒を入れて（「現在」ボタンで再生位置を取り込み）ボタンを押すと、
    `cut_<開始>-<終了>.mp4` を作品フォルダに作り、その場に表示。
  - **⬇ 保存**：動画（や切り抜き）をダウンロード → Finder から X にドラッグ。
- 投稿文・アフィリリンクのコピーも一覧と同じように使える。
- 止めるときは実行中のターミナルで `Ctrl+C`。
- 切り抜きには **ffmpeg** が必要（`brew install ffmpeg`）。
- 再生と保存だけなら `build_board.py <cid>` で作った `board_<cid>.html` を直接開いてもよい
  （切り抜きだけは `serve_board.py` 経由が必要）。

画像の選定・A/B判定をしたいときだけ、旧ボードを使う：

```bash
python3 fanza_auto/_archive/board_v1/serve.py      # 旧ボード（画像まわり担当）
```

実行すると `works/` の下に**未掲載の新作だけ**フォルダが増える（既存はスキップ）。
各フォルダの `投稿内容.md` に沿って、**サブ→メインの引用フロー**で投稿する：

1. **サブ** … 「① サブ投稿」を先に投稿。画像/動画を添付し、アフィリンク付きで出す
2. **メイン** … そのサブ投稿を**引用**して「② メイン投稿」を出す（集客フック）
3. **賑やかし** … 時間をあけてサブ投稿へのリプライで1〜2個だけコメント
4. フォルダ名の `朝` / `夜` がおすすめ投稿時間

## 設定

- 認証情報：プロジェクト直下の `.env`（`DMM_API_ID` / `DMM_AFFILIATE_ID`）から自動で読む。
- 細かい挙動：`config.json` で上書き可能。`config.example.json` をコピーして作る。
  - `fetch_count` … APIからの取得件数（既定50。この中の未使用上位を投稿化）
  - `posts_per_day` … 仕上げる投稿フォルダ数（既定2）
  - `avoid_repeats` … 過去に使った作品を避けるか（既定 true）。使用履歴は `output/_used.json`
    に記録され、翌日以降はランキングの次の作品が選ばれる。同じ日付で再実行した場合は
    その日の分を未使用扱いにして再現する。`false` にすると常にランキング上位を使う
  - `sample_images_per_item` … 公式サンプル画像の取得上限（既定20）
  - `images_per_post` … 1フォルダに入れる画像枚数（既定10／公式＋セーフフレームで充填）
  - `post_slots` … 各投稿のおすすめ時間。フォルダ名に入る（既定 `["夜","朝"]`）
  - `aff_link` … アフィリンクの `af_id` / `ch` / `ch_id` を上書き（リンクの形を変えたい時）
  - `download_movie` … サンプル動画をDLするか（既定 true）
  - `movie_quality` … 画質 `mhb_w`(高/~64MB) / `dm_w`(中/~14MB) / `sm_w`(軽/~6MB)（既定 dm_w）
  - `extract_safe` … 動画から規制セーフな示唆的フレームを抽出するか（既定 true）
  - `safe_frames_top_n` … 抽出する枚数（既定10／画像10枚の充填に使う）
  - `accounts` … 投稿内容.md に表示するアカウント名（@MAIN/@SUB等を実名に）

## 規制セーフなフレーム抽出（extract_safe_frames.py）

サンプル動画から「エロいが X の規制に引っかかりにくい」フレームを自動抽出する。

- `nudenet` で各フレームの露出部位を検出し、**全裸・性器・露骨（Adult/Explicitランク）が写るコマを除外**。
- 着衣・部分露出・チラ見せ・表情など **Sensitiveランクの示唆的フレームだけ**を残す。
- 判定基準は `X運用ナレッジ_アダルト規制.md` に準拠。
- 単体実行も可：`python3 fanza_auto/scripts/extract_safe_frames.py <動画.mp4> [出力先]`

> ⚠️ **自動判定は年齢を見れない。** Xのゼロトレランス（未成年に見える/制服強調）は
> 別途【人間が目視で必ず除外】すること。出力はあくまで一次フィルタ。


## 安全・運用上の注意

- **ブースターは弱め運用**：同一管理者の複数アカウントが毎回群がるのはX最大の凍結要因。
  毎回全部使わない／時間を分散／文面を変える、を徹底する。
- **画像/動画はDMM公式のサンプル素材**を使用（規約・著作権の観点で安全）。
- **X は動画投稿の方がリーチが伸びる**ので、投稿は基本 `sample.mp4` を優先。画像はその補助。
- リンクは**サブ投稿に持たせ、メインはそのサブを引用する**（メイン本体にリンクを貼らない）。
- センシティブ設定は必ずONにする。
- 画像・動画は**候補**。未成年に見える/制服強調のコマは人が目視で必ず除外。

## 生成物

- `works/<cid>_<作品名>/` … 1作品ぶんのフォルダ（cidで一意・差分追加）
  - `01.jpg`〜 … 投稿用の採用画像（規制セーフフレーム優先＋公式サンプル補充）
  - `sample.mp4` … サンプル動画（動画優先で投稿）
  - `item.json` … 作品情報（タイトル・評価・収録時間・発売日・ジャンル・メーカー）
  - `投稿内容.md` … 勝ち型を適用した本文一式（サブ→メイン引用→賑やかし 等）
  - （旧ボードでの操作物）`cut_*.mp4`(動画切り抜き) / `clip_*s.jpg`(画像切り抜き) / `alt_*.jpg`(再選出)
- `works/board.html` … **投稿ボード**（作品情報＋投稿文＋アフィリリンクをコピー／サーバー不要）
- `works/posts.json` … ボードに出す投稿文の保存先。一度作ったら固定される
  （文を作り直すのは `build_board.py --regen` のときだけ）
- `works/index.html` … 旧ダッシュボード（画像の切り抜き・A/B判定はこちら）
- `works/status.json` / `works/verdicts.json` … ステータスと画像A/B判定の保存DB（全体で各1つ）
- `works/00_審査チェックリスト.md` … 最新実行ぶんの源泉除外＋目視チェック項目

## スクリプト早見

| スクリプト | 役割 |
|---|---|
| `fetch_and_build.py` | 新作を取得して `works/` に追加（画像・動画・`item.json`） |
| `meta.py` | 作品情報を取り直して `item.json` を作る（`--all` で全件更新） |
| `post_text.py` | 作品情報から投稿文を1本作る（本文の安全フィルタもここ） |
| `build_board.py` | `works/board.html`（一覧）を作る。cid を渡すと1作品だけの動画つきボード |
| `serve_board.py` | 1作品ボードを動画の切り抜きも使える状態で開く（`<cid>` を渡す） |
| `common.py` | 設定読み込み・アフィリリンク整形・`works/` 走査の共通土台 |
