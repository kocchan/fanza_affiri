# 調査メモ：HTMLダッシュボード → X への画像/動画 投稿（ドラッグ&ドロップ可否）

最終更新：2026-06-23
対象：`fanza_auto/works/index.html`（`serve.py` で配信するダッシュボード）から、X（旧Twitter）の投稿欄へ
画像/動画を**ドラッグ&ドロップで直接添付**できるか。

---

## 結論（先に）

- **ブラウザのHTMLページから X の投稿欄へ D&D で“ファイルとして”添付するのは、実用上できない**（ブラウザのセキュリティ制約）。
- 現実的な最短ルートは次の2つ：
  1. **画像** → ダッシュボードに「📋画像コピー」ボタンを足し、**X で Cmd+V 貼り付け**（最有力・実装も軽い）。
  2. **動画** → ダッシュボードの「保存」で落とした実ファイルを **Finder から X へドラッグ**（既に保存リンクあり。動画はコピペ不可なのでこれが現実解）。

---

## なぜ D&D 直添付ができないのか（技術的理由）

X の投稿欄は、ドロップされたデータの中の **実ファイル（`DataTransfer.files`）** を読んで添付する。
OS（Finder/エクスプローラ）からドラッグしたファイルにはこれが入るので添付できる。

一方で **Web ページ上の `<img>`/`<video>` をドラッグした場合**、`DataTransfer` に入るのは
**画像のURL（`text/uri-list` / `text/html`）であって、ファイル本体ではない**。
→ X 側は「ファイルが無い」と判断し、**添付されない**（URLテキスト扱い or 無視）。

### JS で「ファイルを乗せたドラッグ」を自作できないか？
- `DataTransfer.setData()` は**文字列しか扱えない**（File を渡せない）。
- `dragstart` で `DataTransfer.items.add(file)` を使っても、**別オリジン（x.com）側の `drop` で `.files` として安定して受け取れない**。
  ブラウザはセキュリティ上、Webページが**任意のファイルドラッグを合成すること**を強く制限している。
- Chrome 独自の `DownloadURL`（`setData('DownloadURL', ...)`）は **デスクトップへの“書き出し”用**で、
  他のWebアプリ（X）の dropzone に**ファイルとして渡るわけではない**。
- ⇒ **どのブラウザでも、ページからXへD&Dでファイル添付する確実な方法は無い。**

---

## 現実的な代替案（実装の重さ順）

### ① 画像「コピー」ボタン → X で貼り付け（◎ おすすめ・軽い）
- X の投稿欄は **クリップボードの画像貼り付け（Cmd/Ctrl+V）に対応**。
- ダッシュボードは既に `navigator.clipboard.writeText`（テキストコピー）を実装済み。
  画像は **`navigator.clipboard.write([new ClipboardItem({'image/png': blob})])`** で同様にコピーできる。
- 実装：各画像 figcaption に「📋画像コピー」ボタンを追加 → fetch で blob 取得 → ClipboardItem で書き込み。
  - 注意：Clipboard API は **PNG が確実**（JPG は要 PNG 変換 = canvas 経由）。**HTTPS か localhost 必須**（`serve.py` の localhost ならOK）。
  - **動画はクリップボード貼り付け非対応**（X は動画ペースト不可）＝この手は画像専用。

### ② 動画は「保存」→ Finder からドラッグ（◎ 追加実装ほぼ不要）
- 動画は**ファイルで渡すしかない**。ダッシュボードに既にある **`保存`（download）リンク**で落とす。
- 落としたファイルを **Finder から X の投稿欄へドラッグ**＝確実に添付できる。
- 体験を上げるなら「保存」を押したら**保存先をトースト表示**する程度で十分。

### ③ （参考）X API で直接アップロード（△ 重い・非推奨）
- v2 API + メディアアップロードで**プログラム投稿は可能**だが、**このプロジェクトは「投稿は人が手動」が大原則**。
  自動投稿はアカウント凍結リスク・運用方針に反するので**採らない**。

---

## 推奨

- **まず①（画像コピペ）を実装**：1作品の画像を選ぶ→「📋画像コピー」→Xで貼り付け、が最速になる。
- **動画は②の運用**（保存→Finderドラッグ）で割り切る。UI上に一言ヘルプを添える。
- D&D直添付は**追わない**（ブラウザ制約で工数に見合わない）。
