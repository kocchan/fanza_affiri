# X運用ナレッジ：画像の合否基準（A/Bテストで言語化）

最終更新：2026-06-21

投稿前の画像を「**いい / クリア / NG**」で1枚ずつ判定し、**何がOKで何がダメか**を言葉で蓄積する。
自動フィルタ（`safe_criteria.py`）の取りこぼしを人の目で補正し、最終的に**フィルタ改善の根拠**にする。

関連：[X運用ナレッジ_アダルト規制.md](X運用ナレッジ_アダルト規制.md)（X分類ルールの大元）

> ## 🔑 運用方針（2026-06-21 確定・最優先）
> **NGの本質は「谷間・性行為・裸/露出」**。**制服・学生コスプレそのものはNGではない**
> （明らかに成人で、谷間・性行為・性的接触が無ければ使う）。
> ※ ただし **ロリ/幼児的/ランドセル等“あからさまに子供”はゼロトレランスで別途NG**。
> ※ Xの公式ラインは「制服＝未成年連想で危険」。本運用はそれを理解した上での**運用判断（自己責任）**。

## 判定ラベルの定義

| ラベル | 意味 | 投稿可否 |
|---|---|---|
| **いい** | 集客に使いたい理想形。低露出で“そそる”/可愛い。凍結リスクほぼ無し | ◎ 積極使用 |
| **クリア** | ギリギリOK。センシティブONなら出せるが攻めすぎない | ○ 条件付き使用 |
| **NG** | 出してはいけない。露骨/性行為/未成年連想/下着あからさま 等 | ✕ 不使用 |

## ⛔ 「NG」画像の条件（随時追記）

<!-- 判定でNGになった画像の共通点を言葉で足していく -->
- **性行為そのもの**が写っている（フェラ/挿入/後背位など）。**モザイクがあってもNG**＝自動フィルタは
  モザイク済み性器を検出できずすり抜けるので、**ここは人が必ず弾く**。
- **性的接触**＝着衣でも、男性に胸を触られる等の性的な絡みがあるカットは**危険＝NG**。
- **胸の谷間が見える**＝早期は**NG（危険）**。日常的な構図でも谷間が出ていたら使わない。
- **ロリ/幼児的/ランドセル等“あからさまに子供”**＝ゼロトレランスでNG。
- **性的状況そのものがNG**＝露出が無く女性が着衣でも、**性的なシチュエーション/体勢**はNG。
  例：ベッドで寝た（半裸の）男性に跨る・覆いかぶさる、エステ→手コキの流れ、絡みの体勢。
  → 「写っている部位」だけでなく **“何をしている場面か（文脈）”** で判断する。
- **男性器（モザイクでも）が写る**＝NG。
- ※「制服そのもの」はNGではない（NGの判定軸は谷間・性行為・**性的状況**・裸/露出）。

## 自動フィルタへの反映メモ

### 実装済み（strict / 2026-06-21）
A/Bの学びを `safe_criteria.py` の strict に反映：
- **男性が写る（FACE_MALE / MALE_BREAST_EXPOSED / MALE_GENITALIA_EXPOSED）＝性的状況とみなしフレーム除外**。
  → ひかりさん全NG型・なお02（男に触られる）型を弾ける。
- **谷間（FEMALE_BREAST_COVERED ≥0.50）・下着（GENITALIA_COVERED）・尻（BUTTOCKS_COVERED）＝除外**。→ なお04型を弾ける。
- **女性の顔が無いカットは不採用**（“いい”＝女性ソロ・顔出し）。
- 露骨判定は strict で閾値0.20に敏感化。公式サンプル画像は strict では不使用（性的状況/コラージュ混入のため）。

### 未解決の限界（人の目視が必須）
- **POVのフェラ等“モザイク＋女性ソロ”**は自動で弾けない（性器はモザイクでnudenet未検出、男性も画面外）。
  - モザイク（ピクセル化）検出を試作したが陽性/陰性が分離せず誤検出リスク大 → 見送り。
  - → **`00_審査チェックリスト.md` で人が最終確認**して弾く運用とする。
  - 将来案：学習ベースのモザイク領域検出 or 「口元アップ＋下向き顔」の姿勢推定での減点。
