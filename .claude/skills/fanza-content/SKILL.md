# FANZA コンテンツ生成（システムフェーズ）

このスキルは運用2フェーズのうち**フェーズ①（システムがコンテンツを作る）**だけを実行する。
実際にXへ投稿する**フェーズ②（人の運用）は行わない**（シャドウバン回避のため手動）。

## 前提の確認

実行前に以下を確認する。足りなければユーザーに伝える。

1. プロジェクト直下に `.env` があり `DMM_API_ID` と `DMM_AFFILIATE_ID` が設定されている
2. `fanza_auto/` に `fetch_and_build.py` 等が存在する
3. Python依存（requests / opencv-python(cv2) / nudenet）が入っている

## ★必ず最初に読む：勝ち型ナレッジ

投稿文（メイン集客文）の方向性は **`ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md`** に従う。実行前に必ず読む。
競合実測（11万/5.8万インプ）で裏取りした「伸びる型＝記録/ランキング/速報/ギャップ/断定」を集客文に効かせる。
- スクリプトの集客文プールは `fanza_auto/templates.py`。**文面を調整・追加するときは必ずこのナレッジに沿わせる**
  （記録・速報・断定・ギャップ型を含め、未成年連想ワード/露骨表現は入れない）。

## 写真の源泉（X規制セーフ）の仕組み

判定基準は **`fanza_auto/safe_criteria.py`** に集約（＝`ナレッジ/規制・安全/X運用ナレッジ_アダルト規制.md` のコード版）。2段で厳選する：

1. **作品レベル（源泉）**：genre/タイトルに**未成年連想ワード**（女子校生・制服・学生・セーラー服・ロリ・幼・JK・1●歳 等）を
   含む作品を**丸ごと自動除外**（`screen_minor_risk`）。「美少女」「女子大生」等の成人語は誤除外しない。
2. **フレームレベル**：動画から抽出した“低露出だけどそそる”フレームを**優先**し、各作品**top4だけ**を採用（公式サンプル画像は補充のみ）。`safe_level` で厳しさを切替：
   - **`strict`（立ち上げモード・既定）**：**下着・胸の谷間・尻も避ける**。パンツ/尻のあからさまなカットは除外、谷間は強く減点 → 顔・表情中心の“ふつうに可愛い”カット。露骨判定も敏感（低確信でも弾く）。
   - **`normal`**：着衣の示唆（谷間/チラ見せ）までOK。アカウントにトラストが付いてから切替。
   - 枚数は `images_per_post`（既定4）。

→ 実行ごとに **`works/00_審査チェックリスト.md`**（最新分）を出力（除外作品＋採用作品のgenre＋目視チェック項目）。
**自動は一次フィルタ。最終は人が目視で未成年連想を除外**する旨を必ず伝える。

## 実行手順

1. プロジェクトルート（`fanza_auto` の親）で次を実行する：

   ```bash
   python3 fanza_auto/scripts/fetch_and_build.py
   ```

   - 初回は nudenet のモデルDLで時間がかかることがある。
   - **差分追加**：すでに `works/` にある作品（cid）はスキップし、ランキング上位の**未掲載の新作だけ**を
     `works/<cid>_<作品名>/` として追加する（日付フォルダは作らない）。新規が無ければ「掲載済み」と表示して終わる。
   - 進捗ログ（取得件数・**未成年連想スクリーニング除外数**・既存スキップ数・動画DL・セーフフレーム抽出）をそのまま見せる。

2. 出力先を確認する：
   - 作品フォルダ：`fanza_auto/works/<cid>_<作品名>/`
     （`投稿内容.md` / `NN.jpg`(採用画像) / `sample.mp4`(取れた時) / `item.json`の`description`(商品ページから
     取得したあらすじ・タグ。DMM APIには含まれないため実ブラウザで別途取得。個別ページに参考用として表示のみ、
     投稿文には自動反映しない)）
   - **審査チェックリスト**：`fanza_auto/works/00_審査チェックリスト.md`（最新実行分）
   - **★勝ち型を各投稿に適用**：別枠のランキング集約投稿は作らず、勝ち型ナレッジの“型”を
     各投稿に散らす（`templates.py` の `PATTERN_ROTATION`）。型＝
     **ranking / record（記録）/ flash（速報）/ gap（ギャップ）/ assert（断定）/ empathy（共感）/ question（問いかけ）**。
     - 各 `投稿内容.md` の冒頭に **🏆 型(pattern)** を明記。型に応じて本文・導線を出し分け。
     - **ranking 型**：メイン無リンクで「3位/2位＝有名女優（`famous_actresses.json`）・1位＝この作品を伏せる」→
       リプ①で動画（答え）→ リプ②でアフィリンク（スレ連結）。
     - **その他の型**：サブ（リンク）→ メイン引用（型のフック）→ 賑やかし。
     - 詳細型：`ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md` ⑦）

3. 追加した各作品の `投稿内容.md`（メイン本文・サブのリンク・ブースター案・素材パス・適用した型）を
   要約してユーザーに報告する（今回新規に追加した作品と、既存スキップ数を伝える）。

4. **投稿ボードを更新する**（人の運用フェーズ用・これがメイン）：

   ```bash
   python3 fanza_auto/scripts/schedule_board.py --open
   ```

   - `works/` の未アーカイブ全作品を1枚の **`works/board.html`** に集約する。
     閲覧・コピーだけなら**サーバー不要**（ダブルクリックで開ける）。
   - 各カードに「作品情報（★評価・収録時間・発売日・ジャンル・メーカー）」＋
     「投稿文1本（①サブ→②メイン引用→③賑やかし）」＋「アフィリリンク」＋MissAV確認。すべてコピーボタン付き。
   - 並び順は**レビューの良い順**（件数で重み付け＝★5.00(1件)より★4.75(44件)が上）。
   - 投稿文は **`works/posts.json` に保存されて固定**される。作り直したいときだけ `--regen`（`build_board.py`側）。
   - 作品情報が欠けている場合は `python3 fanza_auto/scripts/meta.py` で取り直す（`--all` で全件更新）。
   - 投稿し終えた作品は「📦 アーカイブ」でボードから消し、`works/archive.html`（右上リンクから移動）へ。
     アーカイブ側では「🗑 完全削除」（フォルダごと消す・元に戻せない）か「↩ 全体ボードに戻す」を選べる。
     アーカイブ状態は `item.json` にサーバー側で保存される（ブラウザのlocalStorageではない）。
   - 切り抜き・アーカイブ・MissAV確認・URL取り込みは `serve_schedule.py` 経由でのみ動く。

   > MyFansの投稿はこのボードには含めない。MyFans専用の別ボード（`myfans_auto/`）を
   > `/myfans-content` スキルで扱う（DMMとMyFansはアフィリエイトの仕組み・投稿元データが
   > 大きく異なるため、意図的にボードを分けている）。

   > ⚠️ 投稿文に出すジャンルは `post_text.py` の `SAFE_GENRES` **許可リスト方式**。
   > 露骨語・未成年連想語は本文に出ない。ただし該当ジャンルが付いた作品はカードに
   > 「画像は要注意」と警告が出るので、**画像は必ず人が目視で確認**する。

   **1作品だけを動画つきで開く**（cid を指定。再生・切り抜き・保存ができる）：

   ```bash
   python3 fanza_auto/scripts/serve_board.py <cid>   # 例: serve_board.py debz015
   ```

   - `works/board_<cid>.html` を作り、`works/` を配信してブラウザで開く（動画のシーク対応）。
   - **✂ 切り抜く**：開始・終了の秒（「現在」で再生位置取り込み）→ `cut_<開始>-<終了>.mp4` を
     作品フォルダに作成し、その場に表示。サーバー（`/__cut`→`cut_video.py`）が処理するので **ffmpeg 必須**。
   - **⬇ 保存**：動画・切り抜きをダウンロード → Finder から X にドラッグ。
   - **②メイン投稿はテキストエリアで直接編集でき、💾保存で`works/posts.json`に保存される**（個別ページ・
     全体ボード両方に反映）。文章を作り直したいときはボード上ではなく、このチャットで直接指示する
     （例：「◯◯のキャプション作り直して」）。
   - 再生と保存だけなら `build_board.py <cid>` で作った HTML を直接開いてもよい（切り抜きは serve_board 経由）。
   - 「◯◯（cid）のボードを開いて」「動画を切り抜きたい」等で使う。止めるときは `Ctrl+C`。

5. **画像を選ぶ・切り抜く時だけ、旧ボードを使う**（アーカイブ済みだが現役で動く）：

   ```bash
   python3 fanza_auto/_archive/board_v1/make_post_html.py   # works/index.html を再生成
   python3 fanza_auto/_archive/board_v1/serve.py            # 旧ボードを開く
   ```

   - 引数なしで `works/` 配下の**全作品**を1枚の `works/index.html` に集約する（日付指定は不要）。
     旧 `output/<日付>` を作りたい場合のみ引数でフォルダを渡せる（後方互換）。
   - 各作品フォルダ（`投稿内容.md` ＋ 画像・動画）をカード化。投稿文はコピーボタン付き、
     画像はクリック拡大＋保存、動画はインラインプレビュー（カット版は枠を強調）。
   - **メディア列のレイアウト**：上から「**サンプル動画（大・元動画）＋切り抜きツール**」→
     「**✂ 切り抜いた素材**（あなたが作成した cut動画・clip画像）」→「**🖼 システムが抽出した画像**」の縦並び。
   - 画像・動画は相対パス参照なので、`works/` 構成は崩さない。
   - 生成後、`open "fanza_auto/works/index.html"` でブラウザ確認を案内する。
   - **投稿ステータス（動的）**：各カードに「投稿前／投稿済／不可」ボタンがあり、変更は
     **`works/status.json`**（ダッシュボード全体で1つ）に保存され再open時も保持される。ただしフォルダ保存は
     **`serve.py` 経由で開いた時のみ**有効（file://直開きはブラウザ内localStorageに保持）。
     - サーバー起動：`python3 fanza_auto/_archive/board_v1/serve.py` → `http://<IP>:8000/`（＝旧ボード）を開く。
     - PC・スマホ同一Wi-Fiでステータス共有可。
   - **動画切り抜き・画像切り抜き（動的）**：サンプル動画下のツールバーで切り抜ける。
     - 🎬 **動画切り抜き**：開始〜終了を入力（または「現在」で動画の再生位置を取り込み）→ `cut_<開始>-<終了>.mp4`。
     - 📷 **画像切り抜き**：秒を入力（または「現在」/空欄で再生位置）→ その瞬間を `clip_<秒>s.jpg` として保存（`grab_frame.py`／ffmpeg必須）。
     - 作成物は「✂ 切り抜いた素材」欄に即追加。`serve.py` 経由でのみ有効。
   - **画像A/B判定（動的）**：各画像の下に **OK／微妙／NG** ボタンと理由入力欄がある。判定は
     **`works/verdicts.json`**（ダッシュボード全体で1つ）に保存され、同時に
     **`ナレッジ/規制・安全/X運用ナレッジ_画像合否基準（ABテスト）.md`** の「判定ログ」表へ
     1行 upsert される（OK→いい／微妙→クリア／NG→NG、同じ画像の再判定は上書き）。`serve.py` 経由でのみナレッジ反映。
   - **OK以外の再選出（動的）**：各画像の「🔄再選出」ボタンで、その作品の `sample.mp4` から
     既存画像と重複しない別の規制セーフ候補を `alt_<連番>.jpg` として追加する（初回だけ候補プール
     `_pool/` を抽出、以降は即時）。追加画像にも判定ボタンが付く。`serve.py` 経由でのみ有効。
   - 元データ（.md・画像・動画）は変更せず、HTML を新規生成するだけ。

## 設定変更（必要なら）

`fanza_auto/config.json` を編集する。主な項目：

- `floor` … `videoc`(素人) / `videoa`(ビデオ) など。`sort: rank` で人気ランキング
- `posts_per_day` … 1回の実行で追加する**新作の上限**（＝既存cidを除いた新規の最大数。既定20）
- `max_new_per_run` … 上限を `posts_per_day` と別に指定したいとき（省略時は `posts_per_day`）
- `movie_quality` … `dm_w`(中) / `mhb_w`(高) / `sm_w`(軽)
- `safe_frames_top_n` … 抽出するセーフ画像の枚数
- `accounts` … シートに表示するアカウント名

特定ジャンルや件数の指定をユーザーが求めたら、config.json を調整してから実行する。

## 完了後に必ず伝えること（重要）

- 出力はあくまで**候補**。人の運用フェーズで以下を必ず行うこと：
  - **未成年に見える/制服強調のコマは目視で除外**（Xのゼロトレランス。`ナレッジ/規制・安全/X運用ナレッジ_アダルト規制.md` 参照）
  - 投稿時に**センシティブ設定をON**にする
  - メインにはリンクを貼らず、リンクはサブのリプライから流す
- このスキルは投稿しない。投稿は人が手動で行う。
