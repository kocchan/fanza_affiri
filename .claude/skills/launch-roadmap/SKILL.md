# 初期アカウント立ち上げロードマップ（コンセプト生成＋HTML）

**実行するたびに新しいアカウントのコンセプトを作り、それを反映したロードマップHTMLを生成する。**
コンセプト固有の中身（表示名・ID・bio・つぶやき例・会話例など）は concept.json に書き出し、
レンダラー（make_launch_roadmap.py）がHTMLに流し込む。手順の地の文・数値は全コンセプト共通。

## 元ナレッジ（実行前に必ず読む）

内容の根拠はこのフォルダ群。文面・手順を変えるときは必ずこれらに沿わせる。
- `ナレッジ/アカウント立ち上げ/X運用ナレッジ_初期立ち上げロードマップ.md`（このスキルの元ネタ）
- `ナレッジ/集客・導線/X運用ナレッジ_アルゴリズム解析（2026）.md`（行動上限・バイラル窓・スコア重みの数理）
- `ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md`（集客文の型）
- `ナレッジ/規制・安全/X運用ナレッジ_アダルト規制.md`（未成年連想/露骨は禁止＝通報回避）
- `ナレッジ/集客・導線/X運用ナレッジ_大喜利（坊主リプ集客）.md`（**大喜利×美女コンセプト**を選ぶときの導線・型）

## 実行手順

1. **元ナレッジを読む**（上の4本。数値・禁止事項を頭に入れる）。
2. **コンセプトの方向性をユーザーに質問する**（`AskUserQuestion`）。すでにユーザーが方向性を明示している項目は飛ばし、足りないものだけ聞く。基本の質問例（2〜4問）：
   - **集客の入口（重要）**：**大喜利リプ集客（坊主などのお題に大喜利で絡む → プロフ流入）**／同ジャンルの伸び投稿にリプ／おまかせ
   - **ジャンル/題材**：**大喜利×美女**（大喜利と美女が好きな人向け）／素人系（ナンパ/ハメ撮り）／企画・単体女優もの／特定ジャンル特化（人妻・ギャル 等）／おまかせ
   - **キャラ・トーン**：目利き・レビュー（信頼の目利き）／共感・あるある（Hあるある・夜の店）／相棒・エモ（深夜の相棒）／煽り・社会的証明（ランキング/速報強め）
   - **見せ方の軸**：当たり作品紹介中心／体験談・語り中心／問いかけ・参加型中心／バランス
   - （任意）**名前の希望や避けたい表現**があれば（「その他」で自由入力）
   - ※各質問は「おまかせ」相当の選択肢も用意し、選ばれたらこちらで提案して埋める。
3. **回答をもとに新しいコンセプトを作る**。
   コンセプトに合わせて以下を**規制セーフ・無リンク・勝ち型**で作文する：
   - 表示名 / @ID候補 / アイコン・ヘッダー方向性 / bio / 固定ポストA,B
   - つぶやき例4本（あるある・問いかけ等）／ リプ定型2本＋問いかけ例
   - **会話例（rally_threads）**：1投稿＝メイン→サブ→メイン…の返信の流れを1〜2本
   - 集客メイン本文2本（記録/速報型）／ サブ本文（リンク差し込み位置つき）
   - **大喜利リプ集客を選んだ場合**は concept.json に `ogiri` を入れる（`target`＝坊主等＋`examples`＝お題→回答を2〜3例）。
     これでフェーズ3が「坊主の大喜利リプで露出を借りる→プロフ流入」に切り替わる。回答は**健全・無リンク**（実リプ文は `/ogiri-answers` で都度生成）。
   - ※**未成年連想ワード・露骨表現・実在名・海賊版は禁止**。題材は明らかに成人。大喜利リプ自体も健全・無リンク。
4. **concept.json を書き出す**（スキーマは `make_launch_roadmap.py` 冒頭のコメント参照）。
   置き場所の例：`ロードマップ/concept_<英数スラッグ>.json`。
5. **HTMLを生成する**（出力先は省略可。省略すると<b>アカウント名入りのファイル名で自動保存</b>）：

   ```bash
   python3 .claude/skills/launch-roadmap/make_launch_roadmap.py <concept.json>
   ```

   - 出力：`ロードマップ/<アカウント名>_立ち上げロードマップ.html`（アカウントごとに別ファイルが残る）。
   - 出力先を変えたいときは第2引数で `.html` パスを渡す。
6. ブラウザで開いて案内：`open "ロードマップ/<アカウント名>_立ち上げロードマップ.html"`。
   - **以後はこのファイルを見ながら進める**（各タスクのチェックはブラウザに自動保存される）。
7. **作ったコンセプト**（表示名・ID・一言）・出力ファイル名・フェーズ/タスク数・重要ポイント（初速＝投稿後2時間の即リプ、会話ラリーの作り方）を要約して報告する。

## 共通の手順・枠を変えたいとき

- フェーズ構成・タスク・注記・CSSなど**コンセプト非依存の部分**は `make_launch_roadmap.py` の
  `build_phases()` / `HTML` を編集する。
- **コンセプト依存の文面**は concept.json 側（または DEFAULT_CONCEPT）で変える。
- どちらも必ず元ナレッジに沿わせ、**未成年連想ワード・露骨表現・メイン直リンクは入れない**。
- 競合分析で新しい勝ち型が出たら `ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md` を更新 → 生成文面にも反映。

## 完了後に必ず伝えること

- これは手順ガイド。**投稿は人が手動で行う**（このスキルは投稿しない）。
- 初期7〜10日は行動量を絞り、外部リンクを本投稿に貼らない（エイジング）。
- 投稿は候補。未成年に見える/制服強調は目視除外・センシティブON。
