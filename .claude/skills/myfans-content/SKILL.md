# MyFans コンテンツ生成・ダッシュボード運用

FANZA（`fanza_auto/`・`/fanza-content`）とは**別の独立したボード**。DMMのようなアフィリエイトAPIが
無い・投稿ごとに個別のアフィリンクが要る、という前提の違いがあるため、意図的に `fanza_auto` とは
切り離してある（`myfans_auto/`）。

## 仕組みの前提（必ず把握しておく）

- URL欄に貼るだけで、`myfans_fetch.py` が自動で以下を取り込む：
  1. **requests**で投稿ページの公開OGPメタタグ（`og:title`/`og:image`）を読む（軽量・ログイン不要）。
  2. **Playwright**で実ページを開き、**年齢確認「はい」だけクリック**して（誰でも通す標準的な導線・
     ボット対策ではない）、本文全文（DOM上の実テキスト。og:descriptionは短く切られているため
     使わない）と、**無料公開されているサンプル動画**（HLS配信・`content.mfcdn.jp`の公開CDN、
     ログイン・購入なしで誰でも見られる）を取得し、ffmpegで `sample.mp4` として保存する。
  3. 2.が何らかの理由で失敗しても、1.の結果（タイトル・サムネ）だけで取り込みを続行する。
- **ログインは自動化しない**。購入後の本編動画などログインが要る情報は取りに行かない方針
  （詳細は下記「試して分かったこと」）。取得する動画はあくまで無料公開のサンプルのみ。
- MyFansのアフィリエイトリンクは**投稿者（クリエイター）ごとに別のコード**が発行される
  （DMMの`af_id`のように口座全体で1つ固定ではない）。取り込むURLは必ず**そのクリエイターのページで
  MyFansが発行した、自分自身のアフィリンク**にする。スクリプト側はURLの中身を検証しないので、
  正しいリンクを貼るかどうかは利用者側の確認に委ねられている。

### 試して分かったこと（ログイン自動化を諦めた経緯）
Playwrightで実ブラウザを操作してログイン→本編動画URLを拾う方式を試したが、ログイン画面の
Cloudflare Turnstile検証が「自動化されたブラウザ」を検知して失敗した。これを回避する技術
（フィンガープリント偽装等）はプラットフォームの意図的な自動化対策を突破する行為にあたるため
実装しない。一方、ログインせず**公開ページを年齢確認だけ通して閲覧する**のは通常の匿名訪問者と
同じ操作であり、Cloudflareに検知されずタイトル全文・本文全文・無料サンプル動画まで取得できる
ことを確認済み。
→ 結論：無料サンプルは自動取得、本編（購入後のフル動画）だけユーザー自身が拡張機能で保存する運用にする。

## 実行手順

1. **① 投稿を取り込む**（タイトル全文・本文全文・サムネ・サンプル動画まで自動）：

   MyFansのボードを開く（切り抜き・URL取り込み・動画取り込みが使える状態で起動）：

   ```bash
   python3 myfans_auto/scripts/serve.py           # works/board.html を開く（既定ポート8000）
   python3 myfans_auto/scripts/serve.py 8100       # ポートを指定（FANZAのボードと同時に開く場合など）
   ```

   - ボード上部のURL欄に、**自分が発行したMyFansアフィリンク**
     （例: `https://link.affiliate.myfans.jp/r/<コード>`）を貼って「＋取り込む」（数秒〜十数秒）。
   - 裏で `myfans_fetch.py` が投稿ページを取得し、`myfans_auto/works/<投稿ID>_<作品名>/` に
     `item.json`・サムネ画像(`01.jpg`)・サンプル動画(`sample.mp4`・取れた場合)・`投稿内容.md` を作る
     （差分追加＝既出の投稿IDはスキップ）。
   - CLI単体でも取り込める：`python3 myfans_auto/scripts/myfans_fetch.py <URL>`
   - 本文の自動抽出に失敗した場合の保険：`myfans_fetch.py <URL> --description "<本文>"` で手動指定できる。

2. **②③（任意）本編の動画を使う**（サンプルで足りるならスキップ）：

   - Chrome拡張機能（FetchV Video Download / ストリームレコーダー等）を使い、いつも通り
     MyFansにログインして該当の投稿を再生 → 動画を保存する。
   - 保存した動画ファイルを、**そのままプロジェクトのルートフォルダ**
     （`myfans_auto/`の親＝リポジトリ直下）に置く。ファイル名はブラウザのタブタイトルのままでよい
     （例：「【〇〇】…タイトル… ◯◯さんのプライベートSNS myfans マイファンズ.mp4」）。
   - ボード上の「🎬 動画を取り込む」ボタンを押す。裏で `import_video.py` が：
     - ルートフォルダの動画ファイル名と、`works/` 内各作品の `item.json` の `title`/`creator` を
       正規化して突き合わせ、**候補が1件に絞れたときだけ**自動で `sample.mp4` として作品フォルダへ
       移動する（①のサンプル動画があれば上書きする）。
     - 候補が複数（あいまい）・0件（未取込のURLが先）のときは移動せず、結果を画面に表示する。
     - CLI単体でも実行できる：`python3 myfans_auto/scripts/import_video.py`

   取り込まれた動画は、以降 `fanza_auto` と同じ切り抜き（✂ 時間で切る）・画面トリミング（🔲 比率/範囲選択）・
   サムネ作成（🖼 動画から切り抜き→候補→トリミング確定）が使える。

3. **④ 投稿作業**：

   各カードに「①リンク投稿（アフィリンク入り）」「②メイン投稿（①を引用）」のコピー欄がある。
   リンク投稿は `fanza_auto/scripts/post_text.py`/`templates.py` を流用。メイン投稿は
   `myfans_auto/scripts/caption.py` が本文（description）から場所・シチュエーション
   （エアビー・お泊まり・ホテル等、行為語やハッシュタグは絶対に使わない許可リスト方式）を拾い、
   「ノリで{語}した結果がこれwww」のような短い反応文にする。該当語が無ければ`templates.py`の
   固定リアクション文にフォールバック。個別ページには本文全文もコピー用ブロックとして表示される。
   投稿し終えたら「📦 アーカイブ」でボードから外す（`works/archive.html` に移る。完全削除／
   全体ボードに戻すも可）。

4. **1作品だけを動画つきで開く**（切り抜き・トリミング・サムネ作成）：

   個別ページはボードの「🎬 動画・画像を編集する →」から開ける。直接指定する場合：

   ```bash
   python3 myfans_auto/scripts/board.py <投稿ID>          # board_<投稿ID>.html を作る
   ```

   切り抜き等の操作自体は①で起動した `serve.py` が処理する（file://直開きは再生・保存のみ）。

## FANZA版との違い（意図的に持たない機能）

- **MissAV確認**：FANZAの品番流出チェック機能。MyFans作品には概念自体が無いので実装していない。
- **評価★・ジャンル・発売日・メーカー表示**：MyFansのitem.jsonには元データが無いため表示しない。
- **未成年連想ジャンルの自動スクリーニング**（`safe_criteria.py`）：MyFansはDMMのgenreデータを持たない
  ため機械的な事前除外ができない。**取り込んだ作品の画像・タイトルは人が目視で確認**すること
  （Xのゼロトレランス方針は`fanza_auto`と同じ）。

## 完了後に必ず伝えること

- 出力はあくまで**候補**。未成年に見える/制服強調のコマは人が目視で除外し、投稿時はセンシティブ設定をON。
- メインにはリンクを貼らず、リンクはサブ（①リンク投稿）から流す。
- このスキルは投稿しない。投稿は人が手動で行う。
