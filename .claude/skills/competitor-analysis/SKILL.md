# 競合分析（バズ投稿スワイプ → 勝ち型へ反映）

競合のバズったX投稿のスクショを解析し、**「どう一言で引いたか（型）」「構成」「実数字」**を抜き出して
スワイプファイル化し、`ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md` に勝ち型として反映する。

> 大原則：**盗むのは“フックの型・構成・形式”だけ。** 題材の危険（制服/学生強調・「脱法ロリ」等の
> 未成年連想ワード・MISSAV等の海賊版参照）は**絶対に盗まない**。うちは正規FANZA＋明らかに成人のみ。

## 入力

- `競合分析/` フォルダ**直下**のスクリーンショット（X投稿のキャプチャ）＝**未処理キュー**。
- ユーザーが「このスクショ」と添付した画像、または同フォルダ直下の未分析分。
- **`競合分析/完了/` 配下は処理済みなので解析しない**（直下に残っているものだけが対象）。

## 分析手順

1. `競合分析/` 配下のスクショを Read（画像）で1枚ずつ見る。
   `_zoom/` に拡大版があれば数字・本文の確認に使う。

2. 各投稿について次を読み取る（読めないものは「不明」と書く。**数字は捏造しない**）：
   - **アカウント構造**：メイン（集客・無リンク）/ サブ（リンク）どちらか。表示名のフック。
   - **本文（キャプション）**：一字一句。
   - **形式**：1枚 / 2枚ギャップ / 4分割 / 動画 / **ランキング** / **スレ（リプ）連結** など。
   - **数字**：インプ（Views）・いいね・RT・ブックマーク・リプ数。
   - **フックの型**：記録/ランキング/速報/ギャップ/断定/共感・自虐/問いかけ/**伏せ（続きはスレ）** 等。
   - **導線**：リンクをどこに置いているか（本文/リプ/引用/スレ）。
   - **規制リスク**：未成年連想・海賊版・露骨フレーム等があれば明記（＝盗まない対象）。

3. **「伸びた型／外した型」を数字で対比**する（インプの桁で判断）。
   伸びた投稿に共通するフック・構成を“勝ちパターン”として言語化する。

4. スワイプファイルを作る／更新する：`競合分析/競合分析_<アカウント名>.md`
   - 既存があれば追記（観測投稿の表に行を足す）。新規アカウントなら新規作成。
   - フォーマットは既存の `競合分析_SNSでは言えない話.md` に倣う
     （アカウント構造 / 観測した投稿と数字の表 / 効いている型 / 真似してはいけない点 /
     うち用の安全テンプレ / 次アクション）。

5. 勝ち型ナレッジ `ナレッジ/コンテンツ/X運用ナレッジ_勝ち型（バズ型）.md` に反映：
   - 新しい型が見つかったら「勝ち型パターン」に追記し、出典スワイプへリンク。
   - **そのまま使える文例（規制セーフ・無リンク）** を、うちの題材（成人明らか・正規FANZA）に
     翻案して足す。未成年連想ワード・露骨表現は入れない。

6. **解析が終わったスクショを「完了」フォルダへ移動**（処理済みと一目で分かるように）：
   - `python3 .claude/skills/competitor-analysis/archive_done.py <今日の日付>`
     （例：`archive_done.py 2026-06-21`）→ 直下のスクショ＋`_zoom/` を
     `競合分析/完了/<日付>/` へ移動する。日付を省くと `完了/` 直下へ。
   - スワイプファイル（`競合分析_*.md`）は成果物なので動かさない。
   - これで次回は**直下に残った未処理分だけ**を解析すればよくなる。

7. ユーザーに要約報告：
   - 今回見つかった勝ち型（特に**新パターン**）
   - 伸びた理由（数字つき）
   - うちの `fanza-content` / `x-tease-posts` にどう移植できるか
   - **完了フォルダへ移動した枚数**（処理済みが分かるように）

## 既知の勝ち型（随時更新）

実測で裏取り済み。詳細は勝ち型ナレッジ参照。
- 記録・ランキングの社会的証明（最強）／速報体／ギャップ2枚／断定・煽り／共感・自虐／問いかけ
- **ランキング＋伏せ＋スレ連結**：「今年バズった〇〇ランキング」で 3位→2位→**1位はさすがに…（伏せる）**。
  好奇心ギャップで本投稿を伸ばし（実測1.1Mインプ級）、**1位の答え・動画・リンクはリプ（スレ）で出す**。
  メイン無リンクの原則と相性が良い（答えを引っ張る＝リプ誘発＆滞在時間↑）。
